5. Display validation errors¶

Separate shell to generate human readable error/warning messages

class rdfa_md.validator_errors.Errors(validator)[source]¶

Shell to generate error and warning messages to the output

Parameters:validator (Validator) – the validator instance that creates this instance

Class variables:

domtree¶

the Document Node of the target DOM tree; originates from the instantiating Validator instance.

target¶

the Element node for the error messages; originates from the instantiating Validator instance.

error_graph¶

the RDFLib Graph to extract the error messages from; originates from the instantiating Validator instance.

validator¶

the Validator instance that has created this instance

Class methods:

_add_element_and_string(parent, element, text, **attrs)[source]¶

Add a DOM element to the parent (unless element == “”) and add a text node to the results

Parameters:
  • parent (Element) – where to add the content
  • element (str) – element name for the new node; if “”, this is skipped
  • text (str) – text to add as a text node
  • attrs – key value pairs for attributes to be added to the new element
_add_string(text, element='p')[source]¶

Add a DOM element to the error block (unless element == “”) and add a text node to the results

Parameters:
  • element (str) – element name for the new node; if “”, this is skipped
  • text (str) – text to add as a text node
header(e, w, i)[source]¶

Generate a header for ‘e’ errors, ‘w’ warnings and ‘i’ information elements. Care is taken to produce a gramatically correct English sentence. The result is added to the DOM tree (via the _add_string() method)

Parameters:
  • e (int) – number of errors
  • w (int) – number of warnings
  • i (int) – number of information elements
one_message(subj, header)[source]¶

Add a single message to the output in the form of a paragraph with <span> elements

Parameters:
  • subj – RDF subject for the error (this is either an RDFLib URIRef or a BNode)
  • header (str) – one of “Error”, “Warning”, or “Info”, added to the final message’s span as a class name, used for CSS
messages(title, msgs, header)[source]¶

Add blocks of messages, preceded by an <h3> element for the title.

Parameters:
  • title (str) – title string for the header
  • msgs (list) – array of error subjects (ie, RDFLib Nodes)
  • header (str) – one of “Error”, “Warning”, or “Info”, added to the final message’s span as a class name, used for CSS
sort_array(arr)[source]¶

Sort the entries of the arrays consiting of subjects of messages. Sorting is based on the time stamp that is added to each of those messages by the RDFLib parser.

Parameters:arr (list) – array of RDF triples
interpret()[source]¶

Interpret the processor graph: separate the warnings, errors, and information elements by message type, and displays the generated message strings in three different categories.
